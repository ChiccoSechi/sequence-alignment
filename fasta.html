FASTA Algorithm

FASTA is a heuristic for fast sequence similarity search. Rather than filling a full dynamic-programming matrix, it first identifies short exact matches between the Query and the Target and uses them as seeds to locate promising regions for a later, more refined alignment.

This page visualizes the first three steps of the algorithm:

  1. K-tuple generation: every substring of length k (the word length, user-defined) is extracted from the Query sequence.
  2. Exact matches in Target: each k-tuple is searched for in the Target. For every occurrence we record posQ (1-indexed position in the Query), posT (1-indexed position in the Target), and the diagonal = posT - posQ.
  3. Diagonals matrix: a Query × Target matrix is built. Each cell where a k-tuple match starts is highlighted, and cells lying on the same diagonal share the same color. Diagonals are then ranked by the number of matches: the top diagonal is a strong candidate region for local alignment.
